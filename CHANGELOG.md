# Changelog

## 0.3.0 Beta

这一版重点修正日期统计口径，并让菜单栏窗口的交互更接近原生 macOS 应用。

### 新增

- 菜单栏窗口内容使用轻量淡入和 4pt 位移动画
- Overview、Models、Activity 和 Logs 页面使用方向感明确的原生切换动画
- 余额、费用、请求数等数字刷新时使用 SwiftUI Numeric Text 过渡
- 刷新状态使用原生 ProgressView 淡入淡出
- 完整支持 macOS“减少动态效果”设置

### 修复

- 修复没有当天调用时，“Today”错误显示最近一次活跃日费用和请求数的问题
- 修复最近 7 天和最近 30 天以最后活跃日为终点、导致旧数据仍被计入的问题
- 修复“最近 7 天请求数”实际统计最后 7 个活跃日期的问题
- 修复“This Month”在 Activity 数据可用时实际显示最近 30 天费用的问题
- 修复费用预测的前后 7 天对比同样锚定最后活跃日的问题
- 打包脚本支持通过变量维护应用版本号和 Build Number

## 0.2.4 Beta

RouterMeter 的首个公开测试版本。

### 新增

- 账户级今日费用，按 Mac 本地自然日计算
- 菜单栏“今日花费 · 剩余余额”组合显示
- Generation 级 Request Logs 页面
- 日志搜索，以及最近时间、费用、Token 排序
- 模型、服务商、Token、费用、延迟、状态、流式和 BYOK 详情
- 最近 100 条日志与增量详情缓存
- 小额费用自适应精度，避免低成本调用显示为零
- 独立应用名称、Bundle ID、Keychain Service 和缓存目录
- 启动时自动刷新

### 修复

- 修复 Analytics 按费用截取少量记录时遗漏低成本模型的问题
- 修复 `gen-stt-*` Generation ID 的时间戳解析
- 修复 Logs 排序控件在窄窗口中换行的问题
